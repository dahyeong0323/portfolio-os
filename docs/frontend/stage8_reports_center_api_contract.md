# Frontend Stage 8 Reports Center API Contract

## Purpose

Stage 8 exposes existing generated Markdown and JSON reports through read-only FastAPI endpoints. The Reports Center is an audit and review surface only. It does not create orders, approve orders, reject orders, execute orders, confirm reconciliations, call brokers, mutate ledger history, or read arbitrary filesystem paths.

## Endpoints

All endpoints are under `/api/v1`.

### `GET /reports/categories`

Returns configured report categories and counts.

Response:

```json
{
  "categories": [
    {
      "category_id": "reconciliation",
      "label": "정산 리포트",
      "description": "Generated reconciliation Markdown and JSON reports.",
      "report_count": 2,
      "supported_formats": ["markdown", "json"],
      "latest_generated_at": "2026-06-14T07:45:00Z"
    }
  ]
}
```

### `GET /reports`

Query parameters:

- `category`: optional configured category id.
- `format`: optional `markdown` or `json`.
- `limit`: optional, clamped to `1..200`.
- `offset`: optional, clamped to zero or greater.

Response:

```json
{
  "count": 1,
  "reports": [
    {
      "report_reference": "report_...",
      "category": "risk_validation",
      "title": "리스크 리포트 #1",
      "format": "markdown",
      "generated_at": "2026-06-14T08:00:00Z",
      "linked_object_type": "risk_validation",
      "linked_object_id": "1",
      "safe_summary": "Risk Validation Report",
      "available_actions": ["view", "copy", "download"],
      "blocked_actions": ["create_order", "approve", "reject", "execute", "confirm", "mutate_ledger", "broker_write"]
    }
  ]
}
```

### `GET /reports/{report_reference}`

Returns one report by an opaque server-generated reference.

Response:

```json
{
  "report_reference": "report_...",
  "category": "reconciliation",
  "title": "정산 리포트 #7",
  "format": "markdown",
  "content": "# Reconciliation Report\n",
  "generated_at": "2026-06-14T07:45:00Z",
  "linked_object_type": "reconciliation",
  "linked_object_id": "7",
  "available_actions": ["view", "copy", "download"],
  "blocked_actions": ["create_order", "approve", "reject", "execute", "confirm", "mutate_ledger", "broker_write"]
}
```

### `GET /reports/{report_reference}/download`

Returns a local file download for a registry-resolved report only. The filename is controlled by the server, and the response does not expose absolute filesystem paths.

## Categories

The registry is limited to known directories:

- `data/exports/reconciliation_reports`
- `data/exports/risk_reports`
- `data/exports/order_tickets`
- `data/exports/senior_memos`
- `data/exports/research_reports`
- `data/exports/macro_reports`
- `data/exports/governance_reports`
- `data/exports/canary_reports`
- `data/exports/context_packages`
- `data/exports/health_reports`
- `docs/frontend` stage Markdown documents

Categories with no files return count zero and empty report lists.

## Safe Reference Rules

- Report references are opaque strings generated by the server.
- The API never accepts raw paths as report identities.
- Filenames are accepted only after decoding a valid server-style reference and must be plain filenames with `.md` or `.json` suffixes.
- Path traversal, absolute paths, unknown references, unsupported formats, and unknown categories return structured errors.
- Report content is returned as text for safe plaintext rendering by the frontend.

## Error Envelope

Errors use the existing API envelope:

```json
{
  "error": {
    "code": "report_not_found",
    "message": "The report was not found.",
    "details": null
  }
}
```

Common codes:

- `report_not_found`
- `unsupported_report_format`
- `unsupported_report_category`
- existing `request_validation_error`
